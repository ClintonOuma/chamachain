Abstract

Problem Statement

  • Manual record-keeping errors & disputes
  • Lack of transparency in fund management
  • Limited credit access for unbanked
  • No fraud detection mechanisms
  • Inefficient meeting coordination

Solution Overview

  • AI-powered credit scoring
  • Blockchain transaction records
  • M-Pesa API integration
  • Role-based access control
  • Real-time notifications
99.9% System Uptime
<500ms AI Response Time
40+ API Endpoints

Objectives

Primary Objective

Develop a scalable, secure, and intelligent digital platform that automates Chama operations while providing AI-driven financial insights and credit accessibility to underbanked populations.

Specific Objectives

1. Digitize Chama operations (contributions, loans, meetings)

2. AI credit scoring using non-traditional data

3. Blockchain transparency for auditability

4. M-Pesa integration for mobile money

5. Fraud prevention with anomaly detection

6. Multi-tenancy with data isolation

Justification

Economic Impact

  • Chamas contribute KES 400+ billion annually
  • 40% of Kenyan adults in savings groups
  • Digital transformation saves 60% admin costs
  • Addresses 4.4M unbanked Kenyans

Innovation

  • First platform combining AI + Blockchain + M-Pesa
  • Addresses UN SDG 1 (No Poverty)
  • Novel credit scoring using group dynamics
  • Creates credit history for unbanked
Market Gap: Existing solutions (M-Changa) lack AI credit assessment. No current platform provides blockchain-verified transaction history with M-Pesa B2C integration.

Limitations

Constraint Impact Mitigation
AI Training Data 6+ months needed for accuracy Iterative model improvement
Blockchain Gas Fees ~$0.001 per transaction Polygon L2 (minimal cost)
M-Pesa Limits KES 150K max per transaction Batch processing for large sums
Internet Dependency Requires stable connection PWA offline mode (planned)
Regulatory CBK compliance required Partner with licensed lenders
